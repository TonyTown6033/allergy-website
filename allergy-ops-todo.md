# 过敏原订单支付与履约 TODO

## 目标

为过敏原检测订单补齐可运营的后台流程，覆盖以下场景：

- 管理员查看支付信息和用户收件信息
- 管理员为已支付订单发货并维护物流状态
- 管理员登记样本回收、检测进度、报告发布和邮件发送
- 后续支持退款、售后和对账

## 账号体系规划

### 当前状态

- 会员端当前是邮箱验证码登录
- 首次登录会自动创建账号
- 还没有标准的“注册 / 登录 / 找回密码”闭环

### 目标方案

- [ ] 支持传统登录方式：`用户名/邮箱 + 密码`
- [ ] 支持独立注册流程，不再依赖“首次登录自动建号”
- [ ] 邮箱验证码仅用于以下场景：
  - 注册时验证邮箱归属
  - 找回密码
  - 高风险安全操作时二次确认
- [ ] 日常登录默认不再要求邮箱验证码
- [ ] 保留后续接入短信、2FA、Passkey 的扩展空间

## 运营临时 SOP

在后台页面完成前，先按下面方式人工处理：

1. 查询 `payment_status=paid` 的订单
2. 查看订单详情，确认收件人、手机号、地址
3. 人工发出采样盒
4. 调用发货接口写入物流公司和物流单号
5. 实验室收到样本后，调用样本签收接口
6. 报告生成后上传 PDF 并发布
7. 如用户没收到邮件，手动补发报告
