# 对账功能文档模板

复制这份模板后，将文件重命名为：

- `<feature>-design.md`

如需细化，再拆：

- `<feature>-api-contract.md`
- `<feature>-test-plan.md`
- `<feature>-migration.md`

---

## 1. 范围

- 本次对账功能覆盖哪些订单、渠道和时间范围
- 哪些财务能力不在当前阶段

## 2. 目标

- 支持谁查看
- 解决什么问题
- 导出给谁使用

## 3. 数据来源

- 订单表
- 支付字段
- 回调原文
- 第三方支付单号

## 4. 视图与筛选

- 时间范围
- 支付渠道
- 订单号
- 用户邮箱
- 支付状态 / 对账状态

## 5. 导出与人工处理

- 导出字段
- 导出格式
- 异常订单人工跟进方式

## 6. 验收标准

- 至少列出 5 条可测试规则
