# 履约备注区与操作日志设计草案

日期：2026-04-15

状态：

- draft

## 1. 范围

这份文档定义订单详情页中的两块后台增强能力：

- 独立备注区
- 操作日志区

不包含：

- 退款逻辑
- 财务对账
- 用户端展示

## 2. 背景

当前订单详情页已经具备履约动作，但备注仍然主要附着在动作请求中的 `remark`，存在几个问题：

- 备注和动作耦合过强
- 无法持续积累运营沟通信息
- 后续追查是谁做了什么不够直观

## 3. 已锁定决策

- UI 放在订单详情页，而不是单独做列表页入口
- 页面中拆成两个区块：
  - 备注区
  - 操作日志区
- 备注和日志不要混在同一个列表里

## 4. 目标能力

### 4.1 备注区

- 管理员可新增备注
- 备注可记录时间、操作人、正文
- 备注不直接驱动状态流转
- 备注用于客服、运营、实验室沟通留痕

### 4.2 操作日志区

- 自动记录关键履约动作
- 至少覆盖：
  - 发货
  - 样本回寄
  - 样本签收
  - 开始检测
  - 上传报告
  - 发布报告
  - 补发邮件
  - 完成订单

## 6. 最低验收标准

- 订单详情页可独立查看备注列表
- 管理员可新增备注
- 备注与状态动作解耦
- 订单详情页可查看操作日志
- 每条关键履约动作都会产生日志
- 日志至少包含动作名、操作人、时间、关联对象
