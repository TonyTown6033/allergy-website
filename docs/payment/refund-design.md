# 退款流程设计草案

日期：2026-04-15

状态：

- draft

## 1. 范围

这份文档定义过敏原订单退款的第一版后台流程。

不包含：

- 自动化财务打款
- 多支付渠道差异化深度适配
- 用户端自助申请退款页面

## 2. 当前问题

- 目前没有退款接口
- 无法区分“已支付未发货”和“已发货”两类退款语义
- 无法沉淀退款状态与退款时间

## 3. 建议先锁定的业务规则

- 已支付但未发货订单，可进入人工退款流程
- 已发货订单，不允许直接走同一条退款快捷路径
- 退款动作必须记录操作人、时间、备注
- 退款结果必须回写订单状态与支付状态

## 4. 待补细节

- 是否需要退款原因枚举
- 是否需要用户通知
- 是否与对账视图联动展示
