# 会员待支付订单取消设计

日期：2026-04-19

状态：

- draft

关联文档：

- [order-cancellation-api-contract.md](./order-cancellation-api-contract.md)

## 1. 目标

为会员端补齐“取消订单”能力，使用户在订单尚未支付、尚未进入履约前，可以主动关闭该订单，避免无效待支付订单长期堆积。

本次改动覆盖：

- 会员端订单详情页增加取消订单入口
- 后端新增会员取消订单接口
- 取消后同步更新订单状态、支付状态和用户可见时间线

## 2. 当前缺口

当前系统只有后台管理员可以把订单更新为 `cancelled`，会员端没有自助取消入口。

这会带来两个问题：

- 用户放弃支付后，订单中心会长期保留 `pending_payment` 订单
- 前台“支付页取消返回”和“用户明确放弃本单”没有区分，订单语义不清晰

## 3. 已锁定决策

- 本功能只允许取消“待支付、未履约”的订单
- 取消订单不等于退款，不处理任何已支付订单
- 支付页回跳 `payment_result=cancelled` 仍只表示“本次支付未完成”，不会自动关单
- 会员取消成功后必须同步更新：
  - `allergy_order.payment_status = cancelled`
  - `allergy_order.order_status = cancelled`
  - `allergy_order.cancelled_at`
  - 用户可见时间线事件“订单已取消”
- 前端不暴露退款、售后、充值等平台或财务语义

## 4. 非目标

- 不处理已支付订单退款
- 不新增后台退款审核流
- 不和第三方支付侧做主动关单联动
- 不新增独立“回收站”或“批量取消”页面

## 5. 状态流转

### 5.1 允许取消

仅当订单满足以下条件时允许会员取消：

- `payment_status = pending`
- `order_status = pending_payment`

取消成功后更新为：

- `payment_status = cancelled`
- `order_status = cancelled`

### 5.2 不允许取消

以下状态不允许会员侧取消：

- 已支付订单
- 已进入采样盒准备、发货、回寄、检测、报告、完成等履约状态的订单
- 已取消订单

### 5.3 支付回跳语义

- 用户从第三方支付页取消返回时，订单仍保持待支付
- 用户只有在订单详情页点击“取消订单”后，订单才会真正进入 `cancelled`

## 6. API 与页面影响

### 6.1 会员端接口

新增：

- `POST /api/orders/:id/cancel`

职责：

- 校验订单归属
- 校验是否仍满足“待支付、未履约”
- 更新订单与支付状态
- 写入用户可见时间线

### 6.2 会员端页面

订单详情页在可取消状态下增加：

- “取消订单”按钮
- 二次确认提示
- 成功后的状态刷新和结果提示

取消后：

- 不再展示支付方式和继续支付按钮
- 改为展示“订单已取消”的状态说明

## 7. 风险点

- 用户曾拉起支付但尚未完成时取消订单，之后如果第三方支付回调迟到，后端不能再把订单更新为 `paid`
- 会员取消只能覆盖未付款订单，不能被误解为售后退款入口
- 前端按钮显示条件必须以后端订单状态为准，不能只依赖 URL 参数或本地临时状态

## 8. 验收标准

- 会员可在待支付订单详情页看到“取消订单”按钮
- 会员取消成功后，订单状态更新为 `cancelled`
- 会员取消成功后，支付状态更新为 `cancelled`
- 会员取消成功后，订单时间线新增“订单已取消”事件
- 已支付或已进入履约状态的订单，会员侧不能取消
- 支付页取消返回不会自动关闭订单，用户仍可继续支付或主动取消
