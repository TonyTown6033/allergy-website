# 支付功能文档模板

复制这份模板后，将文件重命名为：

- `<feature>-design.md`

如需细化，再拆：

- `<feature>-api-contract.md`
- `<feature>-test-plan.md`
- `<feature>-migration.md`

---

## 1. 范围

- 这次支付相关改动覆盖哪些场景
- 哪些支付语义不在本次内

## 2. 已锁定决策

- 支付渠道
- 成功后的业务语义
- 不可复用的旧语义

## 3. 状态流转

- 发起支付前
- 支付中
- 支付成功
- 支付失败
- 退款中 / 退款完成

## 4. 数据与字段

- 复用字段
- 新增字段
- webhook 或回调原文处理

## 5. API / 页面影响

- 用户端接口
- 管理端接口
- 后台设置项

## 6. 风险点

- 重复回调
- 幂等
- 多渠道差异
- 退款边界

## 7. 验收标准

- 至少列出可测试的业务规则
