# 检测项目图片上传设计

日期：2026-04-19

## 目标

为后台“检测项目”管理页补齐本地图片上传能力，使运营在创建或编辑检测项目时，既可以继续手填外链 `image_url`，也可以直接上传本地图片文件并回填为站内可访问地址。

本次改动只解决“检测项目图片上传与展示”这一条链路，不扩展为通用素材中心。

## 当前缺口

当前检测项目目录虽然已有 `image_url` 字段，但后台表单只支持手工输入图片 URL：

- 运营需要先把图片传到外部图床或静态资源目录
- 后台无法直接使用本地文件完成上架配置
- 现有架构文档中也明确了第一阶段先不做图片上传能力，导致当前实现仍停留在 URL-only 模式

## 已锁定决策

- v1 采用服务器本地持久目录存储图片，不引入对象存储
- 后台表单采用“本地上传 + URL 手填并存”
- 数据库结构保持不变，仍只保存最终 `image_url`
- 上传接口单独存在，不把检测项目创建/编辑接口改成 `multipart/form-data`
- 上传成功后返回站内公开图片 URL，由前端回填到检测项目表单
- 前台 `Allergy` 页面继续通过现有图片字段展示，不增加新数据结构

## 非目标

- 不做通用媒体库或图片管理后台
- 不做图片裁剪、压缩、加水印、审核
- 不做图片删除引用追踪或垃圾文件清理
- 不改前台商品列表接口字段名
- 不处理首页 Hero、文章、Testimonials 的图片上传

## 数据与存储

数据库继续复用：

- `allergy_service_product.image_url`

本次不新增表、不新增列。

新增本地文件目录：

- 建议目录：`storage/allergy-product-images`

文件规则：

- 服务端生成唯一文件名，避免覆盖
- 仅允许图片文件上传
- 上传完成后返回站内公开访问路径，例如：
  - `/uploads/allergy-product-images/<file-name>`

## API 与页面影响

### 后台接口

新增：

- `POST /api/admin/service-products/upload-image`

职责：

- 接收管理员上传的单张图片
- 完成基础校验和本地落盘
- 返回最终可提交到 `image_url` 的站内路径

检测项目现有接口保持 JSON 语义不变：

- `POST /api/admin/service-products`
- `PATCH /api/admin/service-products/:id`

### 静态访问

`new-api` 需要暴露公开静态路由，允许前台直接访问上传后的检测项目图片。

### 后台页面

检测项目编辑弹窗新增：

- 本地图片选择与上传按钮
- 上传成功后自动回填 `image_url`
- 已有 `image_url` 时展示图片预览

并保留现有：

- 图片 URL 手填输入框

### 前台页面

前台不改接口契约。

上传后只要 `image_url` 为站内相对路径，前台继续按现有逻辑拼接 API 域名并展示。

## 风险点

- 本地文件需要持久化目录，部署时不能把图片放进易丢失的临时目录
- 公开静态路径必须避免与现有前端静态资源或 API 路由冲突
- 只做新增上传，不自动清理旧图片，后续可能产生未引用文件
- 必须兼容历史外链 URL，不能把已有检测项目图片读写逻辑改坏

## 验收标准

- 后台可在检测项目编辑弹窗中选择本地图片并上传
- 上传成功后，表单中的 `image_url` 自动回填为站内路径
- 管理员仍可手工输入外链 URL
- 创建和编辑检测项目都能使用上传后的 `image_url`
- 上传后的图片可被前台商品页正常展示
- 现有检测项目目录、发布/归档、下单快照链路不受影响
