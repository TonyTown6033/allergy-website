# 账号功能文档模板

复制这份模板后，将文件重命名为：

- `<feature>-design.md`

如需细化，再补：

- `<feature>-api-contract.md`
- `<feature>-test-plan.md`
- `<feature>-migration.md`

---

## 1. 范围

- 这次账号功能改动覆盖哪些场景
- 哪些认证能力不在本次内

## 2. 背景

- 当前现状
- 现有问题
- 为什么现在做

## 3. 已锁定决策

- 登录标识
- 验证方式
- 会员准入边界
- 不做事项

## 4. 数据与状态

- 账号字段
- 会员资料字段
- 会话或验证码用途

## 5. API / 页面影响

- 后端接口
- 前端页面
- 登录态行为

## 6. 风险点

- 老账号兼容
- 邮箱唯一性
- 管理员与会员边界

## 7. 验收标准

- 至少列出 5 条可测试规则
